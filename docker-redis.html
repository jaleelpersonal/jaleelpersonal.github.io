Docker部署Redis

Posted on Thu, Oct 20, 2022 Docker Redis

docker 部署redis 并配置远程连接

1. 拉取 redis 镜像

docker pull docker.io/redis

2. 准备 redis 的配置文件

在 / root/docker/redis/conf 创建好 redis.conf 文件。redis.conf 可以从 redis 官网下载 然后启动的时候导入 redis 的配置文件，就可以按照配置来启动了。

mkdir -p /home/base/redis/conf
mkdir -p /home/base/redis/data
cd /home/base/redis/conf

3. 下载 reids 配置文件

在 conf 目录下运行此命令会直接下载好 redis.conf

wget https://raw.githubusercontent.com/antirez/redis/5.0/redis.conf -O redis.conf

4. redis 配置文件修改

对 redis.conf 配置文件进行修改，我直接把

# bind 127.0.0.1 注掉

protected-mode yes  改成了 no

解释：

requirepass zhangsan2019@ #密码

bind 127.0.0.1    #注释掉

protected-mode no #启用保护模式

5. docker 启动 redis

docker run -d --privileged=true -p 6379:6379 --restart always -v /home/base/redis/conf/redis.conf:/etc/redis/redis.conf -v /home/base/redis/data:/data --name redis redis redis-server /etc/redis/redis.conf --appendonly yes

上面启动是没有采用密码的。直接连接即可

如果临时修改了 redis.conf 文件需要重新加载以下配置文件使用如下命令：

systemctl restart network && systemctl restart docker    #重启network服务

docker exec -ti redis redis-cli -h localhost -p 6379  进入redis

6. 启动命令相关

-d                                                                                 -> 后台运行

-p 6379:6379                                                                       -> 绑定宿主机端口

--name myredis                                                                     -> 指定容器名字

--restart always                                                                   -> 开机启动

--privileged=true                                                                  -> 提升容器内权限

-v /root/docker/redis/conf:/etc/redis/redis.conf                                   -> 映射配置文件

-v /root/docker/redis/date:/date                                                   -> 映射数据目录

  --appendonly yes                                                                 -> 开启数据持久化

7. 由于设置 redis 没有设置密码，而且还作死使用了默认端口 6379，被黑客扫描到了直接给注入了木马程序，强烈建议大家不要使用默认端口，而且设定密码，密码的话启动的时候加上以下命令即可

--requirepass 666666    #666666为密码

端口号我使用了 6399

docker run -d --privileged=true -p 6399:6379 --restart always -v /home/base/redis/conf/redis.conf:/etc/redis/redis.conf -v /home/base/redis/data:/data --name redis redis redis-server /etc/redis/redis.conf --appendonly yes --requirepass 666666

或者修改配置文件, 找到配置文件里面的 requirepass 去掉注释，加上你的密码：

然后启动的时候挂在即可, -v /home/base/redis/conf/redis.conf:/etc/redis/redis.conf 前面 //home/base/redis/conf/redis.conf 是宿主机的路径。

docker run -d --privileged=true -p 6399:6379 --restart always -v /home/base/redis/conf/redis.conf:/etc/redis/redis.conf -v /home/base/redis/data:/data --name redis redis redis-server /etc/redis/redis.conf --appendonly yes
